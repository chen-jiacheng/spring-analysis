# spring-analysis
spring framework source code analysis project.



## 一、Git 提交信息规范
常见的提交前缀:

| 前缀        | 含义 |
|-----------|----|
| feat:     |新功能（feature）    |
| fix:      |修复 bug    |
| docs:     |文档相关的修改    |
| style:    |代码风格的调整，不涉及逻辑修改    |
| refactor: |重构代码，既不修复 bug 也不添加新功能    |
| test:     |增加或修改测试    |
| chore:    |构建过程或辅助工具的变动    |
